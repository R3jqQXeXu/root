LIGGGHTS Documentation

(1.1.5 Version)

LIGGGHTS stands for LAMMPS Improved for General Granular and Granular Heat Transfer Simulations.

The main developer of LIGGGHTS is Christoph Kloss, who can be contacted at christoph.kloss@liggghts.com. Bug reports can be sent to bug@liggghts.com.

www.liggghts.com has more information about the code and its uses. For questions about the code, please use the forums at www.liggghts.com

LIGGGHTS is part of the CFDEM project with the goal to develop a new CFD-DEM code totally based on open source software. The main developers for the CFD-DEM coupling are Christoph Goniva and Christoph Kloss, christoph.goniva@cfdem.com and christoph.kloss@cfdem.com. For more information, please visit www.cfdem.com.

LIGGGHTS is based on LAMMPS (see below), and so is its manual. Many thanks to Steve Plimpton and others at Sandia for providing LAMMPS.



LAMMPS stands for Large-scale Atomic/Molecular Massively Parallel Simulator.

LAMMPS is a classical molecular dynamics simulation code designed to run efficiently on parallel computers. It was developed at Sandia National Laboratories, a US Department of Energy facility, with funding from the DOE. It is an open-source code, distributed freely under the terms of the GNU Public License (GPL).

The primary developers of LAMMPS are Steve Plimpton, Paul Crozier, and Aidan Thompson who can be contacted at sjplimp,pscrozi,athomps at sandia.gov. The LAMMPS WWW Site at http://lammps.sandia.gov has more information about the code and its uses.

The LAMMPS documentation is organized into the following sections. If you find errors or omissions in this manual or have suggestions for useful information to add, please send an email to the developers so we can improve the LAMMPS documentation.

Once you are familiar with LAMMPS, you may want to bookmark this page at Section_commands.html#comm since it gives quick access to documentation for all LAMMPS commands.
